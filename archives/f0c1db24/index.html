文章永久链接

默认的配置的文章url应该类似于这样：https://blog.spranaky.xyz/2020/07/29/hello-world/，这样既不方便搜索引擎的检索，又不方便一些中文标题。如果能让一篇博客文章的url看起来像这样就好了：https://blog.spranaky.xyz/archives/f0c1db24/，这样就确保了一篇文章的url与其内容无关联，无论内容标题怎么变，其永久链接都不会变。当然你选择设不设置都可以，全凭个人喜好。下面来介绍使用这种url的方法。

新文章的创建

创建新的博客文章，是博客系统的一项基本功能，下面我们来看看如何创建新文章：

layout为新建文章的方式，将决定新建的文章的性质及存放位置，如post、page、draft等，是一个可选参数，参数详解：

layout 解释
post 创建一篇新的博客文章
page 新建一个页面，比如网站介绍，友情链接页面等
draft 新建一篇博客文章草稿，可以在写完的时候转为正式文章

在省略参数layout的时候默认为post，可在博客项目/_config.yml中的default_layout中更改。

<title>是要新建文章的标题，不能省略。

创建的草稿文章默认部署之后是不可见的，如果想要强行预览草稿，更改配置根目录下的文件，把render_drafts设置为true，或者在使用hexo s命令的时候加上--drafts参数——hexo s --drafts。至于把草稿变成文章，或者页面的话：hexo publish [layout] <filename>。

对了，hexo默认使用的文章格式为markdown，markdown语法简单，简单学习即可上手，菜鸟教程。这里推荐一款用于编写markdown的超好用的软件——Typora。

基本页面

有的时候我们需要新建一些页面，比如“关于”、“友链”之类的，或者我们想为文章分类或者贴上标签，也需要新建页面。新建页面命令：

比如，我们要新建一个分类页面，我们就要使用hexo new page categories命令，这里categories只是为了说明新建的页面是分类页，事实上，你可以选用任意名字。也可以新建一个标签页面：hexo new page tags。

新建好分类和标签页面之后，就要说明这两个页面是用来对文章进行分类和标签的，hexo新建的页面默认都会在博客项目文件夹下的source文件夹内建立名为<title>的文件夹。所以新建了分类和标签页面之后在source文件夹内就存在有categories和tags文件夹，这两个文件夹内都各有一个index.md，分别打开它们。开头应该会有title和date，title的话，可以改成任意你喜欢的词语，不过为了说明这是一个[分类/标签]页面，最好就写上“分类”和“标签”。

在date参数的下方就没有参数了，事实上，为了准确说明这是一个[分类/标签]页面，你需要再加上一个参数：type，type参数可以标识这个页面的类型，因为我们要建立的是分类和标签页面，所以在两个index.md的后面分别加上type: "categories"和type: "tags"。

但是，你会发现，新建博客的时候每篇博客文章的上面依然只会有title和date，如果要对文章进行分类的话，就要在文章上面的框框内再加上categories，要贴上标签的话，就要追加tags，比如，这篇文章markdown源文件的开头是这样的：

标签可以用类似的语法添加多个，但是一篇文章只能有一个分类。

如果你想添加一个“关于”页面，执行hexo new page about命令就可以了，然后在博客项目/source/about/index.md下就可以进行编辑了，一般新建的页面是不需要分类或贴上标签的，所以也不需要categories和tags参数。

推荐设置

next.yml

自从 NexT-7.3.0 开始，官方推荐使用数据文件将配置和主题分离，这样可以在不修改主题源码的情况下完成主题的配置，便于后续 next 版本更新。

我们将所有的主题配置都放在 博客项目/source/_data/next.yml 中，所有的修改都将在这个文件中进行，不修改主题配置文件 博客项目/themes/next/_config.yml 。

具体步骤：

  • 博客项目/source/下 如果 不存在_data目录，则创建 _data 目录；
  • 将主题配置文件 next/_config.yml 复制到 hexo/source/_data/ 文件夹中，并把文件名从_config.yml更改为next.yml；
  • 把 博客项目/source/_data/next.yml 文件中 override 参数设置为 true 。

valime是一个简洁的评论系统，可以很方便的在next中配置。

由于valine是基于LeanCloud的，因此我们需要注册一个LeanCloud账号才行。注意：如果你的网站是托管在Github Pages等国外服务上的，那么一定要选择LeanCloud的国际版，要不然无法使用（LeanCloud国内版需要网站有备案）。

有必要说明一下valine.visitor参数，设置了这个参数为true之后就不需要设置leancloud_visitors参数了。

另外，valine.guest_info参数默认有nick、mail与link，也就是说，一个访客要评论一篇文章的话，需要在文章最下方的评论框中填入昵称、邮箱与链接（点击评论者的昵称即可访问链接），如果不填写的话昵称默认为Anonymous。valine使用的头像服务是Gravatar，在Gravatar中注册一个账号并且上传头像之后就可以使用头像服务了。在评论框的mail栏中输入你用于注册Gravatar所使用的邮箱账号即可在评论中显示头像。
